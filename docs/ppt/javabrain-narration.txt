# JavaBrain 答辩旁白脚本(剪映 TTS 版)

> 用途:复制到剪映字幕轨,使用"文本朗读"生成配音
> 语速建议:剪映默认 1.0x 即可(约 200 字/分钟)
> 配音建议:剪映"成熟男声"或"知性女声"
> 标点含义:
>   `,` 短停顿(200-300ms)    `。` 长停顿(500-800ms)
>   `?` `!` 带语气    `……` 拉长停顿(1s)    `——` 连接,不停

> **总时长目标**:8 分钟(约 1200-1300 字)
> **每页字数分配**:见每页标题后的`(秒 · 约 N 字)`
> **多音字 / 专有名词处理**:见文末"附录:发音速查表",如 TTS 念错,手动纠音

## 附录:发音速查表(剪映 TTS 念错时手动纠)

> 剪映 TTS 默认会把 "A I" 念成"艾"、"S Q L" 念成"思-扣"等。
> 如果自动念错,可在剪映里:
>   1. 选中该字 → 右键 → "多音字纠错"
>   2. 或换成同音词

| 原词 | 推荐念法 | 备选写法 |
|---|---|---|
| AI | "A I"(读字母) | "人工智能" |
| MCP | "M C P"(读字母) | "模型上下文协议" |
| SQL | "S Q L"(读字母) | "结构化查询语言" |
| CRUD | "C R U D"(读字母) | "增删改查" |
| API | "A P I"(读字母) | "接口" |
| JDK | "J D K"(读字母) | — |
| Qwen | "通义千问" | "Qwen" |
| GPT | "G P T"(读字母) | — |
| RAG | "R A G"(读字母) | "检索增强生成" |
| Skill | "Skill"(读"思高") | "技能" |
| Spring Boot | "Spring Boot"(读"思普林 不特") | "Spring 框架" |
| Amis | "Amis"(读"诶米丝") | — |
| Calcite | "Calcite"(读"卡尔赛特") | "跨库联邦" |
| H2 | "H 2"(读"诶奇-图") | — |
| oms | "o m s"(读"哦-艾姆-艾斯") | "订单管理系统" |
| % | 念"百分之" | "百分号" |
| `——` | 念为短停顿(0.3s) | 改用 `,` |
| `/` | 不念,用作停顿标记 | 改用 `,` 或 `…` |

## 附录:剪映操作步骤

1. **新建剪映工程** → 选 16:9 1920×1080 30fps
2. **导入 PPT 转好的 MP4 骨架**(每页一段)
3. **导入录屏** demo1-final.mp4 和 demo2-final.mp4,**拖到第 6/7 页对应时间段**
4. **添加字幕**:
   - 把本文件每页的 `...` 内容,逐句粘到剪映字幕轨
   - 每句时长与旁白标注秒数一致(如 P1 共 15 秒,字幕 3 行,每行 5 秒)
5. **生成配音**:
   - 选中字幕 → "文本朗读" → 选"成熟男声"
   - 调整语速(默认 1.0x,嫌慢可调 1.1x)
6. **校对发音**:遇到念错的多音字,右键字幕 → "多音字纠错"
7. **导出** → 1920×1080 H.264,比特率建议 8-12 Mbps

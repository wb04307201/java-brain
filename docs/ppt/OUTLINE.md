## 第 3 页 · 定位 + 组成

**定位**（放在页面正中央，字号最大）：

> **JavaBrain —— 企业数字化系统的智能化解决方案**
> 让业务系统快速具备"思考"与"执行"能力

**组成**（3 个组件）：

- 🧠 **灵梭 AI Agent**：Spring AI 编排，零代码接入 RAG / MCP / Skill
- 🛠️ **SQL 工坊**：CRUD + Calcite 联邦查询 + Amis 低代码
- 🔗 **SQL 工坊 MCP**：让 AI 安全对话业务数据库

**为用户提供 3 个智能助手**：

- 💬 **对话智能体**：自然语言交互，智能问答，多轮对话

- 🎨 **智能化低代码助手**：一句话生成 CRUD 页面，10 分钟可用

**组件化特性**：

```text
✓ 3 个组件各自独立，可单独使用也可组合为完整闭环
✓ 按需引入，不捆绑
✓ 组合使用时 = 企业数字化系统的智能化闭环
```

## 第 5 页 · SQL 工坊 + MCP · 4 starter + 6 大功能

- **标题**：SQL 工坊 · 4 个 starter + 6 大功能 + MCP 隔离墙

- **本次演示中的角色**：

  > SQL 工坊 = **数据底座**：搭建 oms，承载订单/商品/用户数据，灵梭通过受限 API 访问，**从不直连数据库**。

- **左上角徽章**：`🧩 独立依赖库 · 4 个 starter · 14 个 Maven 模块 · 可单独引入`

**4 个可选 starter**（顶部一行，强调按需组合）：

```text
✓ sql-forge-spring-boot-starter              必选（基础 CRUD）
✓ sql-forge-calcite-spring-boot-starter       可选（跨库联邦）
✓ sql-forge-web-spring-boot-starter           可选（Amis 低代码 + Console）
✓ sql-forge-mcp                                可选（MCP 工具，本次演示用了它）
```

**6 大功能卡片**（2 行 3 列，★ 标记的 3 个是杀手锏）：

| 图标 | 模块 | 关键能力（一行） |
| :---: | :--- | :--- |
| 🔌 | **JSON CRUD 协议** ★ | `POST /api/json/{method}/{table}` 一套协议覆盖所有表（5 种 method + 16 种 condition） |
| 🌐 | **Calcite 跨库联邦** ★ | 一条 SQL 跨 MySQL + PostgreSQL + H2 跨库 JOIN，**无需 ETL** |
| 🎨 | **Amis 低代码** | JSON 模板 + Web Console；用户/角色/权限管理；**支持模板 save + 预览**（刚演示的就是这个） |
| 🏗️ | **实体 + 模板引擎** | Java 实体 + JPA 注解（@Id / @Table / @Column）+ SQL 模板 + Record 切面 |
| 🔒 | **MCP 5 类受限工具** ★ | 共暴露 25 个 MCP 工具，分 5 类：元数据查询（8，含 findTablesByName / describeSchema）/ SQL 执行（2，**仅 SELECT**）/ JSON CRUD（5）/ Amis 模板（4）/ SQL 模板（6，含安全执行路径） |
| 🛡️ | **MCP 零直连风险** | LLM 不直连 JDBC，走模板化 API；**数据库不直连外网，LLM 可换本地模型实现完全私有化** |

- **底部小字**：`内置用户认证（Session + ApiKey 双模式） · 数据库直连 API 可选开启 · SQL 模板 + Record 切面 · 14 个 Maven 模块`
- **配图建议**：顶部 4 个 starter 拼图（标注"按需引入"）；下方 6 个卡片 2×3 网格，杀手锏金色边框；右下角加 SQL 工坊架构图（**14 个 Maven 模块**，概念上归纳为 8 层：core / entity / record / model / template / web / calcite / mcp）
